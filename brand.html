§ 01
Design Principlesthe four rules everything else descends from.
01

Photography is the primary voice.

Every content-bearing template leads with a hero image. Templates that cover collections rather than subjects (indexes, Contact) use a typographic hero variant (hero-type) — never a plain "page intro" in body text.

02

One focal point per view.

Never two competing hero elements on a screen at once. Stacked hierarchy, never parallel. Breadcrumbs and meta collapse to give the hero room.

03

The logo is locked.

Existing skyline + script mark on black. Not redrawn, not re-tinted, not inverted without an approved adaptation. The system works around it.

04

Editorial numbering, everywhere.

Buildings, listings, and blog entries carry a permanent No. 01 assigned at creation — numbers never reshuffle when inventory changes. See § 09.1.

3.3 — Terracotta Usage Rule

Permitted
Display type (≥20px), italic accent phrases in headings, stats, numbered prefixes (No. 01), decorative rules, focus outlines, link :hover state, backgrounds behind bone text.
Forbidden
Body copy links (default state), paragraph-embedded emphasis, any text under 18px on bone, any text under 14px bold on bone. These ship as WCAG AA violations.

4.4 — Copy Rules

Italic accent rule
One italicized phrase per headline. Always Fraunces italic, always terracotta. Never two in one headline, never a full italic line.
Italic phrase input
Content models carry a separate title_italic field — CMS editors never write raw HTML in titles. Renderer splits: {title.plain} + <em>{title_italic}</em> + {title.tail}. See § 09.
Eyebrow labels
Always uppercase Manrope 600 at 0.22em–0.24em letter-spacing. Exceptions require approval.
Numbers
Listing counts, years, stat headlines use the stat style (Fraunces italic, terracotta). Sequential identifiers (No. 01) use Fraunces regular at eyebrow sizing.

4.5 — Copywriting Guidelines

All copy on this site is written primarily for search engines and AI retrieval systems (Google, Perplexity, ChatGPT, Claude). Secondary readers are humans. The style: plain, specific, how a knowledgeable person would actually write. Not a copywriter performing.

Punctuation — em dashes
Never use em dashes (—) in body copy. They are the single strongest tell of AI-generated writing. Use periods, commas, parentheses, or colons instead. The only permitted em-dash usage is in this spec document itself and in section-title decorative headings.
Punctuation — en dashes
Permitted for number ranges only (2019–2024, 2–3 bedrooms). Never as a sentence break.
Tone
Normal human writing. Informed, direct, specific. Not elegant. Not punchy. Not clever.
Forbidden phrasings
Rhetorical flourishes common to AI output: "It's not just X, it's Y," "In a world where…," "Think of it as…," "The truth is…," "At its core…," "Let's be honest," "Here's the thing." Also forbidden per § 4.4: "stunning," "dream home," "luxury at its finest," "prestigious," "exclusive," "nestled," "boasts," "offers," "features" as a verb.
Sentence structure
Vary length naturally. No stacked three-sentence-each-starting-with-the-same-structure paragraphs. No parallel rhetorical sentences in a row. If two consecutive sentences feel symmetrical, rewrite one.
Specificity rule
Every paragraph must contain at least one concrete, verifiable detail: a building name, floor number, year, unit count, street, amenity, or specific feature. Generic lifestyle copy fails this test and gets cut.
SEO structure
Lead with the noun phrase people actually search for. "Fifth & West condos" before "a building Nicole represents." H1s and first paragraphs contain the primary keyword for the page. Use natural language variants ("condos," "residences," "units") rather than keyword-stuffing a single term.
AI retrieval structure
Write paragraphs that answer a likely question in the first sentence. AI systems extract answer-shaped content. Example: "Fifth & West is a 39-story condo building at 501 West Avenue, completed in 2017." Not: "Rising from the heart of downtown, Fifth & West offers…"
First person
Nicole speaks in first person in blog posts and the About bio. Listing copy and building descriptions use third person (objective, factual). Never "we" — this is a single-agent brand.
Numbers
Use digits for anything countable or measurable: 90 units, 2 bedrooms, 15 years. Spell out only when starting a sentence or when the number is part of a phrase ("a handful of").
Review checklist
Before publishing any body copy, confirm: (1) zero em dashes, (2) no forbidden phrasings above, (3) at least one concrete detail per paragraph, (4) primary keyword in H1 and first 100 characters, (5) no sentence sounds like it was written to be quoted.

§ 06
Motionrestrained, consistent, purposeful.

Motion supports hierarchy; it never performs. If a reader wouldn't notice its absence, it's correct. If an animation draws attention to itself, it's wrong.

6.1 — Tokens

--dur-fast
150ms — button/link color shifts, focus states
--dur-base
240ms — hover image zooms, card lifts
--dur-slow
480ms — page-level reveals, scroll-ins
--ease-out
cubic-bezier(0.16, 1, 0.3, 1) — primary easing for everything
--ease-in-out
cubic-bezier(0.65, 0, 0.35, 1) — gallery transitions only

6.2 — Defined Interactions

Link hover
Color ink → terra over --dur-fast. Underline already present; no new one appears.
Image card hover
Image scales 1.00 → 1.03 over --dur-base with --ease-out. Card itself does not move.
Button hover
Background ink → terra; underline translates Y 0 → Y 2px.
Page load
Hero image fades from 0 → 1 over --dur-slow. Headline translates Y 12px → 0, staggered 80ms.
Scroll reveals
Fade + Y 16px rise on intersection, once. IntersectionObserver, rootMargin: -10%.
Gallery transition
Crossfade --dur-base. No slide, no wipe.
prefers-reduced-motion
All durations → 0ms. Scroll reveals become instant. Hover transforms disabled. Opacity transitions preserved.

10.2 — Delivery Format

Master
TIFF or 16-bit uncompressed. Min 4000px long edge.
Web masters
JPG quality 90, sRGB, 3840px long edge.
Responsive sizes
3840 · 1920 · 1280 · 640. Generated at build.
Formats served
AVIF primary, WebP fallback, JPG legacy. Via <picture>.
Color grade
Warm white-balance, lifted shadows, rolled-off highlights. No HDR blending. No cool corrections. No flash.
Forbidden
Fisheye / ultra-wide distortion, HDR, blown highlights, cool WB, agent headshots against backdrops, stock ATX imagery.

§ 12
Platform Baselineresponsive, a11y, performance.

12.1 — Breakpoints

mobile
< 720px — 4-col grid, 24px side pad, nav collapses to drawer
tablet
720px–1023px — 8-col grid, 40px side pad, nav visible, 2-col card grids
desktop
1024px–1439px — 12-col grid, 56px side pad, full nav, 3-col card grids
wide
≥ 1440px — container capped at 1280px; hero-bleed permitted to extend full viewport
Viewport units
Use svh / dvh for hero heights to handle mobile URL-bar changes. Never raw vh.

12.2 — Accessibility Requirements

Standard
WCAG 2.2 AA minimum. AAA where achievable without compromising design.
Semantic HTML
Single <main> per page. Heading levels increase by at most one. Images have meaningful alt (describe the subject, not "image of").
Focus indicators
:focus-visible { outline: 2px solid var(--terra); outline-offset: 2px; border-radius: 2px; }. Never removed.
Keyboard nav
Full site operable keyboard-only. Tab order follows visual hierarchy. Skip-to-content link on every page (first tab stop).
Interactive cards
Whole-card click surfaces: wrap title in <a>, apply ::after overlay for the click area. Keyboard focus lands on the title anchor, not the container div.
Gallery
Lightbox traps focus, returns to trigger on close. Arrow keys + ESC wired. aria-live="polite" announces "Image X of Y — {caption}".
Forms
All inputs have <label>. Errors announced via aria-describedby. Never color-only error states.
Motion
prefers-reduced-motion honored per § 06.
Text scaling
All sizes in rem. Layout holds at 200% zoom without horizontal scroll.
Contrast
Per § 3.2. Terra text restricted per § 3.3.

12.3 — Performance Targets

LCP
< 2.0s on 4G. Hero images preloaded via <link rel="preload">.
CLS
< 0.05. Every image has explicit width/height.
INP
< 200ms.
Page weight
< 800KB above the fold, images excluded. Hero image budget 300KB AVIF / 500KB JPG fallback.
Fonts
Self-hosted (§ 4.1). font-display: swap. Fraunces variable file (SOFT + opsz axes only).
Gallery loading
First 2 images eager; rest loading="lazy". Lightbox preloads only adjacent (prev + next) on open.
CI gate
Lighthouse CI on every PR. Fails build if Performance or A11y scores drop below 90.
